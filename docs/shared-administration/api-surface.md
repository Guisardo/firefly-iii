# Iteration 1 API surface inventory

This inventory classifies the v1 API for shared-administration work. It is scoped to the route surface in `routes/api.php`.

Classification terms:

- **Supported explicit group**: the endpoint already accepts or resolves `user_group_id` through `ValidatesUserGroupTrait` or a route-level user group parameter and can be called for a non-active administration when the caller has membership.
- **Active-group only**: the endpoint is in or near the Iteration 1 account/transaction surface, but currently scopes through `auth()->user()->user_group_id`, `auth()->user()->userGroup`, or user-owned relations. Keep active-group fallback, but do not treat this as explicit shared-administration support.
- **Global/system**: the endpoint is installation, admin, configuration, user, currency catalog, batch, preference, cron, or otherwise not scoped to a shared administration in the Iteration 1 model.
- **Out of MVP**: the endpoint is a domain surface outside the Iteration 1 account/transaction MVP. Some of these endpoints already have partial explicit-group behavior; keep that noted, but do not expand them in Iteration 1.

## Follow-up checklist

- Budgets and budget limits: route binding, `transactions-without-budget`, budget-limit transaction reads, auto-budget data, export/chart/autocomplete.
- Categories: route binding, no-category reads, transaction attachment/list subresources, export/chart/autocomplete.
- Bills/subscriptions: object groups, rules, transactions, attachments, autocomplete/export.
- Piggy banks: account relationships, events, object groups, transaction-created events, autocomplete/export.
- Rules and rule groups: test/trigger side effects, rule expression validation, rule-owned bill/transaction links.
- Recurrences: trigger side effects, generated transaction ownership, recurrence transaction list.
- Webhooks: trigger-transaction, message/attempt ownership, replay/delete semantics.
- Currencies and exchange rates: per-group default/primary/enabled pivots, currency subresource reads, global admin creation/deletion.
- Reports, charts, insights, summaries, search, and autocomplete: promote only after their underlying repositories/collectors accept resolved group.
- Attachments: attachment ownership must come from attachable object group, not just attachment row user/group fields.
- Exports: verify each export generator uses resolved group before promoting that domain.
- Purge/destroy and bulk APIs: require separate destructive-action review and explicit membership/role tests.
